# Field Notes

`mempool-lab` is easiest to review by starting with the fixture, not the prose.

The domain cases cover `fee pressure`, `mempool churn`, `block fit`, and `replacement margin`. They sit beside the smaller starter fixture so the project has both a compact scoring check and a domain-flavored review check.

The model rewards signal and confidence, then pulls back for drag. In this fixture, `stress` shows where that pullback matters most.

The local verifier covers this data so the notes stay tied to code.
